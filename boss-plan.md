# 보스 몬스터 설계 (boss-plan)

10웨이브 단독 보스전. 주사위 리치 마법술사 컨셉.
9웨이브 클리어 → 10웨이브 진입 시 **코어가 플레이어에 장착**되어 목줄처럼 관성으로 따라온다.
보스는 화면 오른쪽에 고정된 채 계속 코어를 공격하고, 플레이어는 움직여 코어를 피신시키며 보스를 처치한다.

---

## 1. 전투 개요

- **진입 연출**: 9웨이브 종료 → 10웨이브 시작 시 코어를 플레이어에 부착.
- **코어 = 피통**: 보스는 플레이어 본체가 아니라 **코어**를 노린다. 플레이어 HP는 신설하지 않음(기존 Core.Health 재사용).
- **회피 = 이동**: 플레이어가 움직이면 코어가 목줄(leash)로 관성 있게 끌려옴 → 조작으로 탄막·레이저를 피함.
- **보스 위치**: 오른쪽 고정. 이동 없음. 코어를 조준해 패턴 시전.
- **승리 조건**: 보스 처치.

---

## 2. 코어–플레이어 목줄(leash) 메커니즘

지금 `Core`는 `StaticBody2D`(제자리 피통). 10웨이브에선 플레이어를 따라 움직여야 함.

- 10웨이브 진입 시 코어를 "장착" 상태로 전환 → 매 프레임 플레이어 위치를 목표로 관성 추적.
- 관성 추적: `현재속도 → (플레이어 방향으로 가속) → MoveToward`. 목줄 길이(최대 거리) 두면 늘어졌다 당겨지는 느낌.
- 기존 `Core.TakeDamage / Hit / HpChanged / Died` 그대로 사용 → 보스가 `core.Hit(...)` 호출하면 됨.
- 결정 포인트: 코어를 계속 `StaticBody2D`로 두고 위치만 스크립트로 옮길지, 이동체로 바꿀지. (충돌 처리 팀원 영역과 겹칠 수 있음 → 협의)

---

## 3. 보스 본체

`Boss : Monster` 상속으로 Health/TakeDamage(color)/VFX/디버프/애니를 재사용하되,
Core로 행군하는 이동 로직은 쓰지 않는다(오른쪽 고정).

- `_PhysicsProcess`: 이동 없음. `BossPatternController`만 tick.
- `Die()` 오버라이드: 보스 사망 = 승리 처리(라운드 종료/연출 신호).
- 두뇌·주 패턴·주사위는 아래 컨트롤러로 분리(기존 `DebuffController`/`MonsterAnimator` 분리 패턴 동일).

```
Boss : Monster
 ├─ BossPatternController   // 주 패턴(탄막/레이저) 주기 시전 + 3초 주사위 방해
 └─ BossData : MonsterData  // 주기·수치·주사위 가중치 데이터화
```

---

## 4. 보스 주 패턴 (대미지)

### 4-1. 탄막 (부채꼴)
- 보스 기준 코어 방향으로 **부채꼴 범위**에 탄막을 뿌린다.
- 파라미터: 탄 수, 부채꼴 각도, 탄속, 데미지, 발사 주기.
- **슬로우 상태이상 시 탄속 감소**(§6 참조).

### 4-2. 레이저 (상시 예고선 → 직격)
- 보스전 **내내 보스↔코어 사이에 빨간 선이 유지**된다 → "여기로 쏜다"는 전조.
- 발사 직전 선이 **점점 두꺼워지며**(차지) 임계에 도달하면 **직격 레이저 발사(탕)**.
- 예고선은 코어를 계속 따라 그어짐(코어가 움직이면 선도 따라옴).
- 파라미터: 발사 주기, 차지(두꺼워지는) 시간, 조준 방식(발사 순간 코어 위치 고정 vs 추적), 데미지.
- **스턴 상태이상 시 레이저 캔슬**(차지 중이면 취소, 예고선은 다시 얇게 리셋).

> 탄막·레이저는 각각 `IBossPattern` 조각으로 분리 → 패턴 추가/수정이 클래스 단위로 격리됨.

---

## 5. 상태이상 반응 (보스는 디버프를 재해석)

보스는 이동을 안 하므로 기존 디버프 효과를 보스 컨텍스트로 재해석한다.
디버프 적용 경로(HitInfo → 디버프)는 그대로 재사용, **해석만 다름**.

| 상태이상 | 일반 몬스터 | 보스 |
|---|---|---|
| 화상(Fire) | DoT | **그대로** (TakeDamage DoT 재사용) |
| 슬로우(Ice) | 이동속도 감소 | **탄막 탄속 감소** |
| 스턴(Earth) | 정지(`IsStunned`) | **레이저 패턴 캔슬** |

- 슬로우: 보스가 `MoveSpeedMultiplier`(또는 전용 배율)를 읽어 탄막 발사 탄속에 곱함.
- 스턴: `IsStunned` 참일 때 레이저 차지/발사 중단.

---

## 6. 주사위 방해 패턴 (3초 주기, 노데미지)

주 패턴과 별개로 3초마다 주사위를 굴려 **플레이어를 방해**한다(직접 데미지 없음).
1~6 균등, **낮은 확률로 7**.

| 눈 | 패턴 | 내용 | 비고 |
|---|---|---|---|
| 1 | 방패병 소환 | 전방에 방패병 소환 | 기존 `skeleton_shield` + Spawner 소환 재사용 |
| 2 | 지팡이 순서 섞기 | 마법 지팡이 슬롯 순서를 셔플 | 🕓 **후순위** — `Wand.Magics`+`Swap`, `Arsenal.Refresh()` (구현은 쉬움, 나중에) |
| 3 | 탄막 3연발 | 탄막 패턴을 연속 3회 | 주 패턴(탄막) 재사용 |
| 4 | 슬로우(캐릭터) | 플레이어 이동속도 감소 | 🕓 **맨 마지막** — Player 코드 수정 동반(리스크) |
| 5 | 자힐(랜덤) | 보스 자기 회복(랜덤량) | 기존 `Heal()` 재사용, 회복량 랜덤 |
| 6 | 레이저 6발(랜덤) | 전방에 랜덤 방향 레이저 6개 | 주 패턴(레이저) 재사용, 조준 랜덤 |
| 7 | 캐릭터 도움 | 갑자기 공격속도 100배 | 반전 조커. `WandManager` 공속 임시 부스트 |

> **미루는 항목**: 2(지팡이 셔플)·4(캐릭터 슬로우)는 뒤로 뺀다. 특히 4는 `Player` 코드를 건드리므로
> 다른 게 다 안정된 뒤 **맨 마지막**에. 그 전까지 주사위 4/2가 뜨면 임시로 리롤 또는 무해 패턴 대체.

- 방해 패턴 시전 중 다음 주사위 잠금(`_busy`) → 겹침 방지.
- 눈별 가중치·7 등장 확률은 `BossData`로 외부화(밸런싱).

---

## 7. 확인 결과 (착수 전 처리 필요)

착수 전 기존 코드로 검증한 결과:

1. **지팡이 순서 섞기 (주사위 2)** — 🕓 후순위. 구현은 쉬움(`Wand.Swap`+`Arsenal.Refresh`)이나 지금은 미룸.
2. **캐릭터 슬로우 (주사위 4)** — 🕓 **맨 마지막**. `Player.cs`엔 `Speed`만 있어 배율 신설 필요 →
   `Player` 코드 수정 리스크 때문에 다른 게 안정된 뒤 처리.
3. **공격속도 100배 (주사위 7)** — ⚙️ `WandManager.FireCooldown`을 임시로 낮추는 방식.
   `WandManager`에 공속 배율/임시 오버라이드 하나 필요.
4. **코어 이동(목줄)** — `Core`가 `StaticBody2D`라 위치만 스크립트로 추적하거나 이동체로 전환.
   충돌·물리 처리가 팀원 영역과 겹칠 수 있어 협의 필요.

## 8. 코드 재사용 / 신규 매핑

| 필요 | 재사용 | 신규 |
|---|---|---|
| 보스 몸통 | `Monster`(Health/TakeDamage/VFX/디버프/애니) | `Boss : Monster` |
| 주 패턴·주사위 두뇌 | 컨트롤러 분리 패턴 | `BossPatternController`, `IBossPattern` |
| 데이터 | `MonsterData` | `BossData : MonsterData`(주기·수치·주사위 가중치) |
| 코어 이동 | `Core.Hit/TakeDamage/Died` | 목줄 추적 로직 |
| 방패병 소환 | `skeleton_shield` + Spawner | 보스가 호출하는 소환 훅 |
| 자힐 | `Monster.Heal` | 랜덤량 래핑 |
| 슬로우/공속(캐릭터) | — | `Player` 속도 배율, `WandManager` 공속 배율 |
| 상태이상 재해석 | 디버프 적용 경로 | 보스에서 슬로우=탄속·스턴=레이저캔슬 해석 |

## 9. 구현 순서 (verify 포함)

1. **10웨이브 진입 + 코어 목줄** → verify: 9→10 전환 시 코어가 플레이어를 관성으로 따라옴, 이동으로 회피됨.
2. **`Boss : Monster` 골조**(오른쪽 고정, Die=승리 훅) → verify: 10웨이브 단독 등장, 맞으면 죽고 죽으면 승리 신호.
3. **주 패턴 — 레이저**(상시 예고선→차지→코어 직격) → verify: 빨간 선 상시 유지, 발사 전 두꺼워짐, 코어 HP 감소.
4. **주 패턴 — 탄막**(부채꼴) → verify: 부채꼴 탄막 발사, 코어 명중 시 데미지.
5. **상태이상 재해석** → verify: 화상 DoT / 슬로우 시 탄속↓ / 스턴 시 레이저 캔슬·예고선 리셋.
6. **`BossPatternController` 주사위 루프**(로그) → verify: 3초마다 1~7 굴림, 7 저확률, 시전 중 잠금.
7. **주사위 1·3·5·6 구현** → verify: 각 눈 단독 동작(2·4는 제외, 임시 리롤/대체).
8. **`BossData` 수치 외부화 + 밸런싱** → verify: .tres만 바꿔 주기·데미지·가중치 조절.
9. **주사위 7**(공속 100배, `WandManager` 배율) → verify: 발동 시 일시적 초고속 발사.
10. **주사위 2**(지팡이 셔플) → verify: 슬롯 순서 섞이고 UI 갱신.
11. **주사위 4**(캐릭터 슬로우, `Player` 코드 수정) → verify: 플레이어 이동속도 일시 감소.
12. (후순위) 진입·승리 연출.

## 10. 결정 사항

**확정**
- ✅ **착수 방식**: **독립 테스트 씬 우선.** `testcombat.tscn`(Core+Player)에 Boss 인스턴스만 얹어 시작.
  VfxLayer autoload로 VFX 동작, Blackboard null 안전. 복사할 코드 없음.
- ✅ **보스 클래스**: **`Boss : Monster` 상속.** 플레이어 마법이 `MagicNode`의 `List<Monster>`로 타깃을 모으므로,
  보스가 Monster가 아니면 아예 안 맞음 → 상속 필수. (Monster 이동부 훅 분리는 M1에서 최소 처리)
- ✅ **코어 이동**: **`StaticBody2D` 유지 + 위치 스크립트 추적**(관성). 물리 이동체 전환 안 함.

**남은 결정**
- **주사위 7 확률**: "낮은 확률" 구체값 → `BossData` 가중치로 (밸런싱 단계에서).
- **레이저 조준**: 발사 순간 코어 위치 고정 vs 추적. (예고선은 상시 추적, 발사 시점 스냅 여부만 결정)
- **보스 타깃 해석**: 씬에 직접 박힌 보스는 `_Ready`에서 `"core"` 그룹으로 코어를 찾음(Spawner 경유 시 `SetTarget`). 둘 다 지원.
